01

What is this?

An unattended agent — not a dashboard you have to babysit. Every morning at 08:30 Singapore time it polls GDACS, USGS and ReliefWeb, merges duplicate reports of the same event into one, keeps only what clears an impact threshold, and writes a single self-contained HTML page: what happened, where, how bad, and who is affected.

On a calm morning it says so in one line — so silence is never ambiguous between “nothing happened” and “the pipeline broke.” A new Red-level event doesn’t wait for tomorrow; it flashes to the same page within the hour.

02

Why it works this way — and where it’s headed

The feeds are noisy, differently-shaped and SLA-free. They double-report the same quake under different IDs, quietly revise or retract it hours later, and emit far more green-alert noise than signal. There is no low-effort way to get a once-a-day picture that tells a calm world apart from a dead feed. So two decisions shape everything:

Reconciler, not reader

It tracks events over time, because the escalation from Green to Red — an update to something already seen — is usually the alert that matters, not a brand-new item.

The model never wakes it

Change detection and thresholds are deterministic. A language model is only ever asked to phrase what’s already been decided — so scheduled runs stay cheap and trustworthy.

Direction: richer confirmation via ReliefWeb’s full API, model-written assessments for the events that clear the bar, and more hazards as trustworthy sources come online — each dropped in behind the same swappable interface.

Who it’s for, and the decision it speeds up

For anyone who has to stay aware of significant humanitarian disasters without living inside raw feeds:

Duty officers Humanitarian responders Situational-awareness desks Analysts & watch teams
It answers one question you ask every morning:
“Does anything need my attention today?”
One glance answers it — what’s significant, where, how bad, who’s affected — and, just as importantly, whether a quiet screen means a calm world or a broken pipe. Minutes of feed-watching become a single, honest look.
04

What it deliberately does not do

Scope is a decision. These exclusions keep the report honest and worth opening:

  • Hazards without a source in the stack — conflict, epidemics, tsunami warnings, landslides, heatwaves.
  • Feeds beyond GDACS, USGS and ReliefWeb — however useful others might be.
  • Notification channels beyond the dashboard — no email, Slack or outbound RSS. The committed page is the only surface.
  • Intraday alerts below Red level — only a new Red event interrupts the daily rhythm; everything else waits for 08:30.
  • Historical archive or backfill — it tracks the active window, not a research database.
  • Region weighting or attention-as-severity — impact decides, not geography or how many documents an event attracts.
  • Let a model decide what’s reportable — the threshold is deterministic, always.
